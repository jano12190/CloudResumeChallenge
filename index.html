AWS Cloud Resume Challenge

The purpose of this project is to showcase the following skills: frontend development, backend development, serverless architecture, and continous integration/continuous development (CI/CD).


Front-end Development

S3

CloudFront

Route 53

I developed the front end of the using HTML, CSS, and JavaScript. The files were then uploaded to an AWS S3 bucket. I configured a CloudFront distribution, selecting the S3 bucket as the Origin. To secure the site, I enabled HTTPS and configured a custom SSL certificate, which I requested through Certificate Manager, in the Distribution settings. I registered the domain name for the site via the AWS console, and used Route 53 for DNS management, directing the domain to the CloudFront Distribution.


Back-end Development

API Gateway

Lambda

DynamoDB

Several steps were required to create the site's back end, which displays the number of times the website has been viewed. This figure is displayed below. First, I created a DynamoDB table. I created an item with an Attribute Name called "views" and set the value to 0. Next, I configured a Lambda Function written in Python. The function references the table and increases the value of the "views" Attribute by 1 each time the site is viewed. Then, I created an API Gateway and integrated the Lambda function. Finally, I wrote a script that fetches the invoke URL of the API Gateway and displays the output on the webpage.

Serverless Architecture

Servers and EC2 instances weren't necessary for the creation or function of this project. Instead, serverless services such as S3, API Gateway, Lambda, and DynamoDB were utilized.
